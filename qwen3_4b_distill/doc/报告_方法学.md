# 方法学（报告草稿骨架）

## 1. 任务与总体设置

- **学生模型**：Qwen3-4B（instruct，带 thinking + chat 模板），唯一被训模型。
- **教师模型**：Qwen3-8B（主）；加分轴用 Qwen3-14B / 32B-AWQ 做教师强度对比。
- **框架**：verl 0.9（训练 / RL / 评测）；本课题代码负责数据构造、度量、编排、归因。
- **硬件**：2×RTX3090（无 NVLink，共享服务器）。
- **流水线**：`蒸馏数据构造 → 数据度量 → SFT → sft_eval → (GRPO) → 归因`。

## 2. 数据（训练/评测严格分离——高质量课题的前提）

| 角色 | 数据 | 规模 | 用途 |
|---|---|---|---|
| **SEED** | MATH-lighteval train | ~7500 | 蒸馏种子 + GRPO 训练 prompt |
| **EVAL（held-out）** | OlymMATH en-hard | 100 | **只评测，绝不进训练**；所有分数报在它上 |

- 统一转成 verl RL parquet（`prompt` / `reward_model.ground_truth` / `extra_info`）。
- ground_truth 归一：MATH 从 `solution` 抽 `\boxed`，OlymMATH 用 `answer` 列。
- `extra_info` 保留 level/type/subject 等，供 §6 按切片归因。
- **泄漏防护**：GRPO/OPD 训练 prompt 只用 SEED(MATH)，olymmath 仅作监控，绝不进训练集。

## 3. 评测协议（本课题的关键方法学决定）

- **指标**：pass@1（avg@n）、pass@k（无偏估计，Chen et al. 2021）；thinking 模式默认开。
- **判定**：verl 内置 math-verify（造数据过滤 / eval / GRPO reward **三处同一判定**，标准一致）。
- **生成预算 = 38912 token（全项目统一）**。依据：
  - Qwen3-4B 在 OlymMATH-hard 思维链中位 ~20K token；预算 8192 时 **96% 被截断** → pass@1 假性归零（实测 2%）。
  - 38912 = Qwen3 竞赛数学官方推荐、且 `+2048` prompt 正好顶满 `max_position_embeddings=40960`；再高需 YaRN（改变模型、破坏可比性）。
  - **base / SFT / GRPO 三方施加完全相同预算**，比较才内部有效。
- **透明度**：`summary.json` 自带 `truncated_rate` / `mean_new_tokens`，每次 eval 都能看出分数是否被预算压过。
- **工程**：满预算 eval 单模型占满两卡 → 交错分片（各卡半数题）后 `merge_shards.py` 合并，指标与单卡等价。
- **[待填]** base（M1 锚点）：pass@1=`__` / pass@8=`__` / 截断率=`__`（与论文 Qwen3-4B≈13.9% 对齐性讨论）。

## 4. 蒸馏方法（公平对比铁律：同一教师 / 同一模板 / 同一采样预算）

- **standard_cot（基线）**：教师直接对种子题生成 CoT，math-verify **拒绝采样**——只留"未截断 + 含 `\boxed` + 答对"的完整推理。
- **reverse（RevThink）**：正向推理 R_f → 由答案造逆向问题 Q_b → 逆向推理 R_b → 一致性过滤，产多目标样本 (Q→R_f, Q→Q_b, Q_b→R_b)。
- **质量门槛统一**：三法都丢弃**教师被截断**的候选（截断 CoT = 半截/无答案，是训练集系统性偏差），并落 `gen_stats.json`（良率 / 截断率 / 长度分位）。
- **[待填]** 各法：良率=`__` / 截断率=`__` / 产出条数=`__`。
- （question_aug 为可选第三法，Xwin-Math 造新题 + self-consistency 过滤；有余量再做。）

## 5. 训练（off-policy 序列蒸馏 SFT）

- Qwen3-4B 全参 SFT，FSDP 跨 2 卡；对教师 CoT **整段（含 `<think>` 思维链 + 最终答案）算 loss**（学的是完整推理过程）。
- **长度**：`max_length` 覆盖数据 `tok_p99/max`，避免长 CoT 被截断丢答案（超长样本预删而非截断）。
- **加速（不改数值，纯提速）**：flash-attn 变长打包（`use_remove_padding`，预编译轮子绕 glibc）+ Liger（SwiGLU/RMSNorm/RoPE）+ verl 融合 linear cross-entropy（省 15 万词表 logits 显存）。
- 超参：lr=1e-5，epochs=3（`[待填]` 如调整）。

## 6. 度量与归因（课题核心："解释 accuracy 为什么变"）

**数据侧度量**（student 视角，`data_metrics.py`）：长度(token) / 多样性(distinct-1,2) / PPL / **IFD**（L(A|Q)/L(A)，越高越难跟随/越像噪声）。

**归因分析**（2 种方法下，统计单位是题 n=100，扎实）：
- **配对切片对比**（`slice_eval.py --vs`）：reverse 相对 standard，按 level/type/subject **逐切片 Δ准确率** + **McNemar 混淆**（谁修好/弄坏了哪些题）+ 错例题面 → 定位"在哪类题上、以什么方式"提升。
- **数据属性 ↔ 表现**（`compare_methods.py` / `attribution.py`）：并排数据度量表；跨方法标量相关系数需 ≥3 点，2 方法时以配对切片为主，相关性为辅。
- **[待填]** 结论：reverse 在 `__` 类题上 Δ=`__`；净修好 `__` 题、弄坏 `__` 题；数据侧差异（长度/多样性/IFD）与提升的对应关系。
