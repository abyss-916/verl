## 2. 任务二：实现 ≥3 种蒸馏方法 + 分析数据特点

> 原文核心：**"不急着训练，先观察不同方法产出的数据有什么差异"**；公平对比；可复现。

### 三种方法实现（原文 §3.1）
| 方法 | 实现(`generate_cot.py`) | 论文对齐 | 状态 |
|---|---|---|---|
| **Standard CoT**(baseline) | 教师直接 CoT + math-verify 拒绝采样 + 完成度过滤(未截断/含\boxed/答对) | — | ✅代码 |
| **Reverse Thinking**(RevThink) | R_f→逆问题 Q_b(I_bq)→逆推理 R_b→一致性过滤(I_con)→三目标样本(Q→R_f, Q→Q_b, Q_b→R_b) | arXiv 2411.19865，`任务二_方法规格.md` 逐 prompt 拆解 | ✅代码 |
| **Question Aug**(Xwin-Math) | Prompt1 造全新题→Prompt2 造解→self-consistency 多数投票过滤(无 gold) | arXiv 2403.04706，规格书含"扩题≫扩解"机理 | ✅代码 |

### 数据分析（任务二真正核心）
| 要求 | 实现 | 状态 |
|---|---|---|
| **先看数据差异**(3 法都分析) | **3 法都造数据**(standard/reverse ~2000；qaug 只分析~500) → `data_metrics`(长度/distinct-1,2/PPL/IFD) → `compare_methods` 对比表 | 📋排期 |
| 公平对比铁律 | 同一教师(Qwen3-8B)/同一 chat 模板/同一预算(38912)/同一 student——代码硬编码保证 | ✅ |
| 可复现记录 | 方法名/teacher/采样/过滤/来源全参数化 + `gen_stats.json` 留痕 | ✅ |
| 下游验证 | **Standard-CoT SFT + sft_eval(base vs SFT)**；reverse SFT+eval 作向上叠加 | 📋排期 |

**训练侧高质量保证**：SFT 对教师 CoT **整段(含 `<think>` 思维链)算 loss**(已查 verl 源码证实)；`max_length` 覆盖数据 p99 避免长 CoT 被截断丢答案。
