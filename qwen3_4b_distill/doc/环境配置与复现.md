# 环境配置与复现（Qwen3-4B 蒸馏课题）

> 记录本课题的**软硬件环境、从零复现步骤、关键决策与踩坑**。既供复现，也作报告方法学章节的素材。
> 一句话总结：在一台**共享的 2×RTX3090（glibc 2.31）服务器**上，把 verl 0.9 的 RL/训练/评测栈跑通，
> 全部产出钉到 `/data/liujiachen`，并用**预编译轮子绕开 flash-attn 的 glibc 障碍**。

---

## 1. 硬件与系统约束（决定了后面所有取舍）

| 项 | 值 | 影响 |
|---|---|---|
| GPU | **2× RTX 3090 24GB，无 NVLink** | 跨卡走 PCIe，TP=2 有通信开销；单卡能装下的模型优先单卡 |
| 归属 | **共享服务器**（多人共用） | 不动系统/全局配置；跑实验前先看 `nvidia-smi` 挑空闲卡 |
| 驱动 / CUDA | 570.86.15 / CUDA 12.8 | 决定 torch 走 cu128 |
| **glibc** | **2.31**（Ubuntu 20.04 世代） | **flash-attn 官方轮子要 GLIBC_2.32，装不上**（见 §4） |
| 系统盘 `/` | **仅 25G** | 所有缓存/临时/下载**必须重定向到 `/data`**，否则爆盘 |
| 数据盘 | `/data`（共用）；**我们的目录是 `/data/liujiachen`** | 一切产出放 `/data/liujiachen`，`/data` 根下是公共区不乱放 |

**两条铁律（每次给服务器指令都遵守）：**
1. 先 `source run/env.sh`（把缓存/临时/产出全钉到 `/data/liujiachen`）；
2. 跑 GPU 实验前先看 `nvidia-smi`，按空闲情况定 `CUDA_VISIBLE_DEVICES` 与 `--gpu_mem`。

## 2. 软件栈（一套自洽的版本组合）

| 组件 | 版本 | 备注 |
|---|---|---|
| Python | 3.12 | conda env 名 `verl` |
| PyTorch | **2.9.0+cu128** | 由 vLLM 依赖链决定 |

## 4. flash-attn：glibc 2.31 的绕行（本课题最典型的坑）

**问题**：官方 PyPI 的 flash-attn 轮子在 manylinux（较新 glibc）上编，import 时报
`ImportError: /lib/x86_64-linux-gnu/libc.so.6: version 'GLIBC_2.32' not found`——本机 glibc 只有 2.31。

**为什么不升级 glibc**：glibc 是系统级、影响所有用户，共享机上升级可能**把整台机器搞崩**，坚决不动。

**为什么不从源码编译**：flash-attn 源码编译要 1–3 小时 + CUDA toolkit，成本高、有风险。

**解决**：用第三方预编译轮子仓库 **[mjun0812/flash-attention-prebuild-wheels](https://github.com/mjun0812/flash-attention-prebuild-wheels)**，
其 **v0.7.0 起用 `manylinux_2_28` 平台编译（要求 glibc ≥ 2.28）**，本机 2.31 ≥ 2.28 → 兼容。
选用 **v0.9.0** 的 `flash_attn-2.8.3+cu128torch2.9-cp312-cp312-linux_x86_64.whl`（241MB，精确匹配 torch2.9+cu128+py3.12）。
国内下载慢 → 本地电脑下好 + xftp 传到 `$TMPDIR` 再 `pip install --no-deps`。
验证 `import flash_attn` 与 `from flash_attn.bert_padding import pad_input` 不报 GLIBC 即成功。

**用途**：仅**训练侧**（SFT/GRPO 前向反向）——开 verl 的 `use_remove_padding`（变长打包，省 padding，长短不一时 1.5–2×）。
eval / 造数据走 vLLM 自带注意力后端，**不依赖**这个 pip 包。

## 5. 训练加速（三件套，均不改数值结果、纯提速）

| 开关 | 依赖 | 作用 |
|---|---|---|
| `use_remove_padding=true`（+ `pad_mode=no_padding`） | flash-attn（§4） | 变长打包，省 padding，1.5–2× |
| `model.use_liger=true` | `pip install liger-kernel`（Triton，不编译） | SwiGLU/RMSNorm/RoPE 融合，~10–20% |
| `model.use_fused_kernels=true` | verl 自带（Triton，无需装） | 融合 linear cross-entropy，省 Qwen3 **15 万词表**的 `[seq×15万]` logits 显存 |
| （回退）`attn_implementation=sdpa` | 无 | 万一 flash-attn 算子跑挂，`USE_FLASH=0` 回退；sdpa 在 Ampere 本就是 FlashAttention-2 内核 |

`sft.sh` 已封装 `USE_FLASH`（默认 1 走变长打包，=0 回退 sdpa+padding），`USE_LIGER`/`USE_FUSED` 可单独关。

## 6. 缓存/产出重定向（`run/env.sh`，防系统盘爆 + 不污染共用区）

`env.sh` 把以下全部钉到 `/data/liujiachen`（**不写 `~/.cache`、`/tmp`、不改 `~/.bashrc`、不动系统**）：

- 产出：`MODELS` / `DATA` / `CKPT` / `LOGS`（模型/数据/ckpt/日志）
- HF：`HF_HOME` / `HF_HUB_CACHE` / `HF_DATASETS_CACHE`；`HF_ENDPOINT=hf-mirror.com`（国内镜像）
- ModelScope：`MODELSCOPE_CACHE`
- torch/triton/vllm：`XDG_CACHE_HOME` / `TORCH_HOME` / `TRITON_CACHE_DIR` / `VLLM_CACHE_ROOT` / `TORCHINDUCTOR_CACHE_DIR`
- pip：`PIP_CACHE_DIR`（默认 `~/.cache/pip` 在系统盘！）；`PIP_INDEX_URL=清华镜像`
- 临时：`TMPDIR` / `RAY_TMPDIR`（避开 `/tmp`）
- wandb：`WANDB_DIR`，默认 `WANDB_MODE=offline`

> 踩坑：曾因没 source env.sh，pip 装 torch 时缓存写到系统盘 → `[Errno 28] No space left on device`。

## 8. 评测预算的方法学决定（写进报告）

- Qwen3-4B 在 OlymMATH-hard 上思维链**中位 ~20K token**；`--max_new 8192` 时 **96% 被截断** → pass@1 假性归零（2%）。
- 定**全项目统一评测预算 `max_new=38912`**（= Qwen3 竞赛数学官方推荐、且 `+2048` prompt 正好顶满
  `max_position_embeddings=40960`；再高需 YaRN，会改变模型、破坏可比性）。base/SFT/GRPO **三方同一预算**才可比。
- `eval_math.py` 的 `summary.json` 自带 `truncated_rate` / `mean_new_tokens`，任何一次 eval 都能一眼看出分数是否被预算压过。
- 满预算 eval 单模型占满两卡 → 用**交错分片**（`--num_shards 2`，各卡跑一半题）+ `merge_shards.py` 合并。

## 附：常见故障速查

| 症状 | 原因 | 处理 |
|---|---|---|
| `GLIBC_2.32 not found` | flash-attn 官方轮子 | 用 §4 的预编译轮子 |
| `No space left on device` | 没 source env.sh，缓存写系统盘 | `source run/env.sh` 后重试 |
| vLLM warmup 时 OOM | 与他人共卡、`gpu_mem` 太高 | 看 `nvidia-smi`，调低 `--gpu_mem`（如 0.7）或换空闲卡 |
| `VLLM::EngineCore` 杀不掉 | vLLM 改了子进程名，`pkill -f 脚本名` 匹配不到 | 按 PID/进程组 `kill`（`nvidia-smi` 查占卡 PID） |
| SFT `ImportError: flash_attn.bert_padding` | flash-attn 没装 / 算子跑挂 | 装 §4 轮子；或 `USE_FLASH=0` 回退 sdpa |
